## 2. Bugs documentados e corrigidos nos scripts Python

### Bug 1 — inject_images offset (+1 posição)
**Arquivo:** `scripts/rewrite_drafts_v2.py`, função `inject_images()`
**Sintoma:** Imagem de anime X aparecia sob o título do anime X+1.
**Causa:** Loop iniciava em `h2_ends[1:]` (pulava H2[0]) e Nones eram comprimidos por filter.
**Fix:**
```python
section_medias = medias[1:]   # skip featured, mantém alinhamento posicional
for i, h2_end in enumerate(h2_ends):   # começa em H2[0]
    media = section_medias[i]          # None = sem imagem, sem shift
    if not media: continue
```

### Bug 2 — Logos em vez de cenas (cartoons ocidentais)
**Causa:** `get_section_bytes()` tentava `cover` antes de `banner`. No AniList, `cover` de cartoons ocidentais é logo vetorial pequena (< 20KB).
**Fix:** Ordem trocada para `banner → cover`. Imagens < 20KB rejeitadas.

### Bug 3 — Featured image errada (hentai)
**Causa:** Script pegava banner do PRIMEIRO show da lista (Urotsukidoji = demônio).
**Fix:** Campo `featured_show_override` no TOPICS:
```python
772: { "featured_show_override": "Agent Aika", ... }
```

### Bug 4 — Slug com sufixo -2
**Causa:** WordPress reserva slug por período após force-delete.
**Fix:** Após publicar, script verifica slug retornado e faz PATCH se diferente.

### Bug 5 — User-Agent "Radiata/2.0" bloqueado pelo Fandom/Wikipedia
**Fix:** `HTTP` session usa Chrome UA:
`Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 Chrome/125.0.0.0 Safari/537.36`

### Bug 6 — Subdomínios Fandom errados
**Subdomínios confirmados (NUNCA usar os incorretos):**
| Show | Correto | Errado |
|------|---------|--------|
| Xiaolin Showdown | `xiaolinpedia` | `xiaolin` |
| Winx Club | `winx` | `winxclub` |
| Static Shock | `static` | `dcau` |
| Dexter's Lab | `dexterslaboratory` | `dexters-lab` |
| Johnny Bravo | `johnnybravo` + page `"Johnny Bravo (character)"` | main page retorna vazio |

**Regra:** Usar páginas de PERSONAGEM (ex: `Dexter`, `Bloom`, `Omi`) em vez de main page do show — infobox de personagem sempre tem imagem direta.

### Bug 7 — Fandom não estava no pipeline principal
**Fix:** `get_section_bytes()` em `rewrite_drafts_v2.py` agora segue:
`AniList banner → AniList cover → Fandom wiki → Wikipedia`
`FANDOM_WIKIS` dict com 26 shows verificados, buscado por `show_name` (PT) e `search_title` (EN).

### Bug 8 — Wikipedia retorna imagem não relacionada (Recess)
**Causa:** `pageimages` API retornou imagem de infobox do artigo (computador/playground).
**Fix:** Para cartoons ocidentais, tentar variantes com sufixo:
```python
wiki_variants += [f"{search_title} cartoon", f"{search_title} TV series characters"]
```
Detecção: se título NÃO contém palavras-chave de anime (naruto, bleach, jujutsu…) → ocidental → adiciona sufixos.

### Bug 9 — Filename sem SEO
**Antes:** `radiata-{slug}-{idx:02d}.jpg` (genérico, sem nome do anime)
**Depois:**
- Featured: `{post-slug}.jpg`
- Seção: `{anime-kebab-slug}.jpg` via `_to_seo_slug(show_name)`

1. `fetch` fire-and-forget sem `keepalive` → browser cancelava ao navegar

**`app/api/dashboard/trigger/route.ts`** — streaming NDJSON:
```

Última linha: { "done": true, "status": "success" }
```

**`components/pipeline-controls.tsx`** — lê 1ª linha do stream:
```typescript
const reader = res.body.getReader();
const { value } = await reader.read();
const { runId } = JSON.parse(firstLine);
router.push(`/logs?run_id=${runId}`);  // navega com run_id correto
```

**`lib/pipeline/orchestrator.ts`** — callback `onRunCreated`:
```typescript
export function runRssPipeline(supabase, options, callbacks?) {
  // Calls callbacks.onRunCreated(runId) as soon as run row is created
}
```

**Visual:** botão mostra spinner `animate-spin` enquanto aguarda o runId (< 1s).

## 5. Sistema 2 — Temas Manuais (novo)

### Como usar
1. Abrir `radiatastories.vercel.app/temas`
2. Colar URL de qualquer artigo de anime
3. Clicar "▶ Reescrever e Publicar"
4. Acompanhar progresso no terminal ao vivo
5. Quando concluir: link para o post publicado aparece

### Pipeline de temas (`app/api/temas/run/route.ts`)
```
Stage 1 🔍 Extract    — extractArticle(url) — texto + imagens
Stage 2 ✍️ Rewrite   — Claude Sonnet 4.6 — PT-BR, 1300+ palavras
Stage 3 🔬 QA        — Claude Haiku — 6 critérios (aprovação não bloqueia post manual)
Stage 4 🖼️ Images   — Upload para WP com nomes SEO
Stage 5 🚀 Publish   — publishPost() → WordPress
```

### Diferenças vs Sistema 1 (RSS)
| | Sistema 1 (RSS) | Sistema 2 (Temas) |
|---|---|---|
| Fonte | 20 feeds RSS | URL manual |
| Trigger | Automático (cron) | Manual (dashboard) |
| QA falha | Pula candidato | Publica mesmo assim |
| Imagens | Da notícia original | Da URL colada |
| system_type | `rss` | `manual` |

### Arquivos criados
- `app/temas/page.tsx` — página com input + terminal ao vivo
- `app/api/temas/run/route.ts` — pipeline streaming NDJSON

## 8. Padrão de imagens (resumo executivo)

1. AniList banner (landscape, > 20KB)
2. AniList cover (> 20KB)
